# Mission 044 - 0.1.3-dev.18 - Vue groupee Midjourney par job

## Choix UI

Choix retenu : toggle dans la vue centrale `Web / Midjourney`.

Le selecteur affiche :

- `Images` : vue plate actuelle ;
- `Jobs` : nouvelle vue groupee.

La colonne gauche n'a pas ete surchargee avec une entree supplementaire.

## API IPC / preload

Aucune nouvelle API IPC n'a ete ajoutee.

La vue groupee reutilise les images Midjourney deja chargees par :

```txt
remote-image:list provider=midjourney
```

Chaque image contient deja :

- `imageId`
- `remoteProvider`
- `remoteProviderGroupId`
- `remoteSlot`
- `imageSrc`

Le renderer ne lit pas SQLite directement.

## Structure des groupes Midjourney

Structure calculee cote renderer :

```txt
MidjourneyJobGroup
- jobId
- images
- slots
```

`slots` est une map logique vers les slots :

- `0_0`
- `0_1`
- `0_2`
- `0_3`

Chaque slot pointe vers un `ImageFile` existant.

## Logique de regroupement

Le regroupement est fait par :

```txt
image.remoteProviderGroupId
```

Si cet identifiant manque, la vue utilise un fallback local sur `displayName`, uniquement pour eviter une carte impossible a rendre.

Les groupes sont tries par `imageId` maximum decroissant, ce qui donne en pratique les jobs les plus recents en premier avec le modele actuel.

## Affichage des slots

Chaque carte job affiche :

- un job ID tronque ;
- un compteur `N/4 images` ;
- une grille 2 x 2 ;
- les slots `0_0`, `0_1`, `0_2`, `0_3`.

## Jobs incomplets

Si un slot n'existe pas dans les images disponibles, la carte affiche une case sobre :

```txt
Slot manquant
```

Aucune image fictive n'est creee.

## Interactions sur slot

Chaque slot disponible reste une image individuelle :

- clic : selectionne l'image ;
- double-clic : ouvre l'image dans la visionneuse ;
- clic droit : reutilise le menu contextuel image existant ;
- panneau droit et metadonnees restent lies a l'image selectionnee.

## Interactions sur job

Chaque carte job propose :

- `Selectionner les images du job`

Cette action selectionne les `imageId` disponibles du job et place le premier slot disponible comme image courante.

La selection peut ensuite etre utilisee avec :

- `CommandOrControl+Shift+C` ;
- le menu `Edition > Ajouter la selection a une collection...` ;
- le clic droit image ;
- le drag vers collection.

## Drag de job vers collection

Le drag de carte job est implemente.

Comportement :

- glisser une carte job vers une collection ;
- les images disponibles du job deviennent la selection ;
- `draggedImageIds` contient les `imageId` des slots disponibles ;
- le drop reutilise le flux existant `collections:add-images`.

Le `dataTransfer` ne contient pas de chemins locaux ni d'URLs distantes brutes.
