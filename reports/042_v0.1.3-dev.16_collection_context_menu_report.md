## Approche du menu contextuel collection

Approche retenue : menu contextuel React/CSS cote renderer, sur le modele du menu contextuel image de la mission 041.

Le clic droit sur une collection dans la colonne gauche ouvre un menu compact avec :

- `Renommer la collection...`
- `Supprimer la collection...`

Le menu se ferme :

- au clic exterieur ;
- au clic droit exterieur ;
- avec `Echap` ;
- apres choix d'une action ;
- lors d'un changement de vue.

## IPC / preload ajoutes

Canaux ajoutes :

- `collections:rename`
- `collections:delete`

API preload ajoutee :

- `renameCollection({ collectionId, name })`
- `deleteCollection({ collectionId })`

Le renderer ne lit ni n'ecrit directement dans SQLite.

## Logique de renommage

Cote main process, `renameCollection` valide :

- existence de la base SQLite ;
- `collectionId` positif ;
- collection existante ;
- nom non vide apres trim et normalisation des espaces ;
- absence d'une autre collection portant deja ce nom.

En succes :

- `collections.name` est mis a jour ;
- `updated_at` est mis a jour ;
- la collection est relue ;
- la base est persistee ;
- la collection relue est retournee au renderer.

Cote UI :

- une modale pre-remplit le nom actuel ;
- `Entrer` valide ;
- `Echap`, `Fermer` ou `Annuler` ferment la modale ;
- un nom vide affiche `Le nom de collection ne peut pas etre vide.`;
- un doublon affiche `Une collection porte deja ce nom.`;
- la liste des collections est rechargee apres succes ;
- si la collection renommee etait active, elle reste active.

## Logique de suppression

Cote main process, `deleteCollection` valide :

- existence de la base SQLite ;
- `collectionId` positif ;
- collection existante.

La suppression est faite dans une transaction :

```sql
DELETE FROM collection_images WHERE collection_id = ?
DELETE FROM collections WHERE id = ?
```

La suppression explicite de `collection_images` est volontaire, meme si le schema contient `ON DELETE CASCADE`, afin de garder le comportement lisible et robuste.

## Texte de confirmation

La confirmation affiche :

```txt
Supprimer la collection "Nom" ?

Cette action supprimera uniquement la collection virtuelle et ses associations.
Les images, fichiers originaux et metadonnees ne seront pas supprimes.
```

Le bouton de confirmation est libelle `Supprimer la collection`.

## Rafraichissement UI

Apres renommage :

- `loadCollections()` recharge la liste ;
- la collection active est remplacee par la version renommee si elle etait selectionnee ;
- le compteur est conserve via la relecture SQL.

Apres suppression :

- `loadCollections()` recharge la liste ;
- si la collection supprimee n'etait pas active, la vue courante reste stable ;
- si la collection supprimee etait active, l'UI bascule vers la premiere collection restante ;
- s'il ne reste aucune collection, l'UI revient a la vue locale ;
- la selection image, l'ancre de selection et la visionneuse sont videes pour eviter une selection fantome.

## Garantie de suppression non destructive

La suppression ne touche qu'a :

- `collection_images`
- `collections`

Elle ne supprime pas :

- `images`
- `local_images`
- `remote_images`
- `image_user_meta`
- `image_terms`
- `image_tags`
- les fichiers locaux originaux
- les images distantes

La mission n'ecrit pas dans les dossiers photo.
