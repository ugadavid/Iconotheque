# Mission 040 - 0.1.3-dev.14 - Correction creation collections

## Resume

Mission realisee depuis `0.1.3-dev.13`, avec passage en `0.1.3-dev.14`.

Objectif : corriger le retour immediat apres creation de collection et ameliorer les messages d'ajout d'images a une collection.

## Fichiers modifies

- `package.json`
- `package-lock.json`
- `CHANGELOG.md`
- `electron/main.ts`
- `src/App.tsx`

## Cause exacte du message de relecture

Le message `Collection creee mais impossible a relire.` venait de l'ordre des operations dans `createCollection`.

Apres l'`INSERT`, le code appelait `persistDatabaseFile()` avant de recuperer `last_insert_rowid()`.

Cette persistance peut recreer ou remplacer l'instance SQLite en memoire. Dans ce cas, `last_insert_rowid()` n'est plus fiable pour relire immediatement la collection creee, meme si l'ecriture a bien ete sauvegardee.

Resultat observe : la collection existait apres redemarrage, mais le retour IPC immediat echouait.

## Correction appliquee

Dans `electron/main.ts`, l'ordre a ete modifie :

- insertion de la collection ;
- recuperation immediate de `last_insert_rowid()` ;
- relecture de la collection par id ;
- persistance de la base ;
- retour IPC avec la collection relue.

Le cas `Collection creee mais impossible a relire.` reste conserve pour un vrai cas exceptionnel, mais ne doit plus se produire dans le flux normal.

## Rafraichissement de la liste

Dans `src/App.tsx`, `loadCollections()` retourne maintenant la liste fraichement relue.

Apres creation :

- le renderer recharge la liste des collections ;
- retrouve la collection creee dans cette liste par `id`, puis par `name` en fallback ;
- ferme la modale ;
- selectionne la collection creee ou relue.

Ce flux permet a la colonne gauche et aux compteurs d'etre mis a jour sans redemarrage.

## Messages d'ajout ameliores

Les messages de la modale `Ajouter a une collection` sont maintenant differencies :

- `Image ajoutee a la collection.`
- `N images ajoutees a la collection.`
- `Cette image est deja dans la collection.`
- `Les images selectionnees sont deja dans la collection.`
- `Ajout termine : X image(s) ajoutee(s), Y deja presente(s).` pour les ajouts partiels.

Le ton visuel est aussi ajuste :

- succes quand toutes les images selectionnees ont ete ajoutees ;
- avertissement quand il y a doublon ou ajout partiel.
