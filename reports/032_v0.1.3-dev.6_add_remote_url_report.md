# Mission 032 - 0.1.3-dev.6 - Premiere image web generique visible

## Resume de la mission

Mission realisee depuis `0.1.3-dev.5`, avec passage en `0.1.3-dev.6`.

Objectif atteint : ajouter un premier parcours visible permettant d'ajouter une image web generique par URL HTTPS, de l'afficher dans une vue minimale `Web / Images par URL`, et de conserver les metadonnees utilisateur via l'identite stable `images.id`.

## Choix techniques

- Conservation du modele SQLite existant `images` + `remote_images`, sans migration de schema.
- Utilisation de `images.id` comme identite stable commune local / distant.
- Ajout de `imageSrc` cote renderer pour afficher local et distant sans exposer de `file://`.
- Conservation de `previewUrl` pour compatibilite interne pendant la transition.
- Selection renderer basculee de `path` vers `imageId`.
- Images distantes sans faux chemin local : `path` vaut `null`.
- Validation URL cote main process uniquement :
  - URL non vide ;
  - protocole `https://` uniquement ;
  - extensions visibles limitees a `.jpg`, `.jpeg`, `.png`, `.webp`, `.gif`.
- Affichage distant via protocole controle `iconotheque-image://remote/<imageId>`.
- Aucune URL distante brute n'est utilisee comme source principale d'image dans le renderer.

## Fonctionnement ajoute

- Menu `Fichier > Ajouter une image web...`.
- Modale renderer minimale pour saisir une URL HTTPS.
- IPC minimal :
  - `remote-image:add-from-url` ;
  - `remote-image:list` ;
  - evenement `remote-image:add-requested`.
- Creation d'une ligne `images.source_kind = 'remote'`.
- Creation d'une ligne `remote_images` avec :
  - `provider = generic_url` ;
  - `source_status = remote`.
- Detection des doublons par URL normalisee exacte.
- Vue gauche minimale `Web / Images par URL`.
- Liste persistante des images web apres redemarrage.
- Affichage dans la grille avec badge `Web`.
- Selection, visionneuse, panneau d'informations et metadonnees compatibles avec les images web.

## Points non traites

- Pas de cache distant.
- Pas d'archive locale.
- Pas de telechargement persistant.
- Pas d'integration Midjourney.
- Pas d'import par lot.
- Pas de racine Web avancee avec collections multiples.
- Pas de recherche avancee etendue aux images distantes.
- Pas de verification reseau au moment de l'ajout, seulement lors de l'affichage.

## Risques / limites

- Une image web peut etre ajoutee meme si elle est temporairement indisponible ; l'erreur apparaitra au rendu.
- L'affichage distant depend du reseau et du serveur tiers.
- Le type `ImageFile` conserve encore `previewUrl` pour compatibilite, a nettoyer progressivement.
- Les recherches avancees restent volontairement centrees sur les images locales autorisees.
- Le protocole distant ne met rien en cache ; les grandes images distantes peuvent etre rechargees.

## Recommandations pour la suite

- Ajouter une indication d'erreur visuelle plus explicite quand une image distante ne se charge pas.
- Etendre prudemment la recherche aux images distantes si le modele UI est valide.
- Ajouter une collection Web plus structuree avant Midjourney.
- Prevoir plus tard un statut `cached` / `archived` uniquement apres cadrage clair du stockage local.
- Nettoyer progressivement les anciens champs de compatibilite `previewUrl` lorsque tous les composants utilisent `imageSrc`.
