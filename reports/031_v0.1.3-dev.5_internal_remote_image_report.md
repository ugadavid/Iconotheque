# Mission 031 - 0.1.3-dev.5 - Premier parcours distant interne

## Resume

Mission realisee depuis `0.1.3-dev.4`, avec passage en `0.1.3-dev.5`.

Objectif atteint : ajouter un parcours technique interne permettant de creer, relire et supprimer une image distante de test dans la base, sans UI publique, sans import URL utilisateur, sans racine Web, sans affichage distant, sans Midjourney et sans acces reseau.

## Validation URL retenue

La validation est syntaxique et prudente.

Regles appliquees :

- URL non vide obligatoire ;
- parsing via `new URL(...)` ;
- seul `https://` est accepte ;
- `file://`, `data:`, `javascript:`, `blob:` et les autres schemes sont refuses par la restriction `https:`;
- si une extension est visible dans le chemin, elle doit faire partie des extensions image supportees :
  - `.jpg`
  - `.jpeg`
  - `.png`
  - `.webp`
  - `.gif`

Aucune requete reseau n'est effectuee.

L'existence reelle de l'image distante n'est pas verifiee.

## Creation distante interne

`createRemoteImageRecord` :

1. valide l'URL ;
2. ouvre une transaction SQLite ;
3. cree une ligne `images` avec `source_kind = 'remote'` ;
4. recupere le nouvel `images.id` ;
5. cree la ligne correspondante dans `remote_images` ;
6. commit ;
7. persiste la base.

Fournisseur utilise par defaut :

```txt
generic_url
```

Aucune ligne `local_images` n'est creee.

## Protection transactionnelle

L'insertion est transactionnelle :

- `BEGIN TRANSACTION`
- insertion `images`
- insertion `remote_images`
- `COMMIT`

En cas d'erreur :

- `ROLLBACK`
- pas de ligne `images` distante orpheline conservee volontairement.

## Relecture par imageId

`getRemoteImageRecordById` relit l'image distante avec :

```sql
images
INNER JOIN remote_images ON remote_images.image_id = images.id
```

La fonction ne retourne un enregistrement que si :

- `images.id` correspond ;
- `images.source_kind = 'remote'` ;
- `remote_images.image_id` existe.

## Suppression de test

`deleteRemoteImageRecordForDev` :

1. verifie que l'image est bien distante ;
2. ouvre une transaction ;
3. supprime `remote_images` ;
4. supprime par prudence d'eventuelles metadonnees / tags / termes rattaches ;
5. supprime `images` avec `source_kind = 'remote'` ;
6. commit ;
7. persiste la base.

La fonction ne supprime pas de fichier image original.

## Auto-test

Auto-test activable uniquement par variable d'environnement :

```txt
ICONOTHEQUE_REMOTE_SELFTEST=1
```

Comportement :

1. initialise la base ;
2. cree une image distante de test :

```txt
https://example.com/iconotheque-remote-test.png
```

3. verifie :
   - `images` contient 1 ligne pour cet `imageId` ;
   - `remote_images` contient 1 ligne pour cet `imageId` ;
   - `local_images` contient 0 ligne pour cet `imageId` ;
4. relit l'enregistrement distant ;
5. supprime l'enregistrement de test ;
6. verifie que les compteurs reviennent a 0 pour cet `imageId` ;
7. quitte proprement l'application.

Resultat observe :

```txt
[Iconotheque] Remote self-test OK: imageId=1781, created={"images":1,"remoteImages":1,"localImages":0}, deleted={"images":0,"remoteImages":0,"localImages":0}
```
