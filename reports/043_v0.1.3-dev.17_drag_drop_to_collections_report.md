# Mission 043 - 0.1.3-dev.17 - Drag and drop vers collections

## Resume

Mission realisee depuis `0.1.3-dev.16`, avec passage en `0.1.3-dev.17`.

Objectif : permettre de glisser une ou plusieurs images depuis la grille vers une collection virtuelle dans la colonne gauche.

## Fichiers modifies

- `package.json`
- `package-lock.json`
- `CHANGELOG.md`
- `src/App.tsx`
- `src/components/ImageGrid.tsx`
- `src/components/FolderTree.tsx`
- `src/components/StatusBar.tsx`
- `src/styles/app.css`

## Approche technique

Implementation React cote renderer avec les evenements drag/drop natifs :

- `draggable` sur les tuiles d'image ;
- `onDragStart` et `onDragEnd` dans `ImageGrid` ;
- `onDragEnter`, `onDragLeave`, `onDragOver` et `onDrop` sur les collections dans `FolderTree` ;
- etat React `draggedImageIds` dans `App` ;
- etat React `dropTargetCollectionId` pour le feedback visuel.

Aucun nouveau canal IPC n'a ete ajoute.

## Initialisation du drag

Au debut du drag, `ImageGrid` appelle `onImageDragStart(image)`.

Cote `App` :

- si l'image glissee est deja dans la selection, le drag porte sur toute la selection ;
- si l'image glissee n'est pas dans la selection, la selection devient cette image seule ;
- `draggedImageIds` est rempli avec les `images.id` concernes ;
- les menus contextuels ouverts sont fermes.

Le `dataTransfer` ne contient pas de chemins locaux ni d'URLs distantes.

Il contient seulement :

- le type symbolique `application/x-iconotheque-images` ;
- un texte neutre avec le nombre d'images.

## Selection multiple

La selection multiple est preservee quand le drag commence depuis une image deja selectionnee.

Le drag d'une image hors selection remplace la selection par cette image seule, comme le clic droit ajoute en mission 041.

## Collections comme cibles de drop

Seuls les boutons de collections dans `FolderTree` acceptent le drop.

Les zones suivantes ne gerent pas le drop :

- dossiers locaux ;
- `Web / Images par URL` ;
- `Web / Midjourney` ;
- titres de section ;
- zones vides de la colonne gauche.

## Drops invalides

Si aucune image n'est en cours de drag, les handlers de collection n'appellent pas `preventDefault`.

Les zones non collection n'ont pas de handler de drop et ignorent donc le depot.

## Ajout a collection

Le drop appelle l'API preload existante :

```txt
collections:add-images
```

Le renderer transmet uniquement :

- `collectionId`
- `imageIds`

Le main process conserve la logique existante :

- validation de la collection ;
- validation des images par `images.id` ;
- insertion uniquement des couples absents dans `collection_images` ;
- dedoublonnage des associations deja presentes.

## Rafraichissement des compteurs

Apres un drop reussi :

- `loadCollections()` recharge les compteurs de la colonne gauche ;
- si la collection cible est la vue active, `loadCollectionImages(collection)` recharge la grille.

## Feedback visuel pendant le survol

La collection survolee recoit la classe CSS `folder-row-drop-target`.

Le rendu ajoute :

- fond legerement accentue ;
- bordure discrete ;
- compteur recolore ;
- ombre interne sobre.

## Feedback apres drop

Le resultat du drop est affiche temporairement dans la barre de statut.

Les messages reutilisent le format stabilise en `0.1.3-dev.14` :

- `Image ajoutee a la collection.`
- `N images ajoutees a la collection.`
- `Cette image est deja dans la collection.`
- `Les images selectionnees sont deja dans la collection.`
- `Ajout termine : X image(s) ajoutee(s), Y deja presente(s).`

Le message disparait automatiquement apres quelques secondes.

## Recette manuelle

La recette interactive complete n'a pas ete effectuee dans cette session.

Non verifie manuellement :

- glisser une image locale vers une collection ;
- glisser une image Midjourney vers une collection ;
- glisser une image web generique vers une collection ;
- glisser une selection multiple ;
- glisser une image hors selection pendant une multi-selection ;
- drop de doublon ;
- drop partiel ;
- glisser depuis une collection vers une autre ;
- tentative de drop sur les sources Web.

## Limites restantes

- Pas de reorganisation manuelle des images dans une collection.
- Pas de deplacement entre collections : le drop ajoute seulement une association.
- Pas de drag and drop depuis l'explorateur Windows.
- Pas de drag and drop de fichiers reels.
- Pas de sous-collections.
- Pas de tri manuel.
- Pas d'export.
- Pas de cache, archive ou telechargement.

## Confirmation fichiers originaux

Aucun fichier image original n'est modifie, copie, deplace, renomme ou supprime.

Le drag and drop ajoute uniquement des associations virtuelles `collection_images.collection_id -> images.id`.
