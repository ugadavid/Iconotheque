# Mission 035 - 0.1.3-dev.9 - Import Midjourney par lot

## Resume

Mission realisee depuis `0.1.3-dev.8`, avec passage en `0.1.3-dev.9`.

Objectif : ajouter un import Midjourney par lot simple, permettant de coller plusieurs job IDs ou URLs CDN Midjourney et de creer ou reutiliser les quatre slots distants de chaque job valide.

La modale reste volontairement minimale et ne remplace pas l'ajout Midjourney unitaire existant.

## Formats acceptes

Chaque ligne non vide peut contenir :

- un job ID UUID Midjourney ;
- une URL `https://cdn.midjourney.com/<job_id>/<slot>.png` avec slot `0_0`, `0_1`, `0_2` ou `0_3`.

Les lignes vides sont ignorees.

Les lignes invalides sont refusees cote main process, notamment :

- UUID invalide ;
- domaine non Midjourney ;
- protocole autre que HTTPS ;
- slot invalide ;
- chemin ne correspondant pas au format attendu.

## Logique de parsing ligne par ligne

Le renderer envoie le texte brut a `window.iconotheque.addMidjourneyJobsBatch(input)`.

Le main process :

1. decoupe le texte par lignes ;
2. ignore les lignes vides ;
3. valide chaque ligne avec la meme logique que l'import Midjourney unitaire ;
4. extrait le `jobId` normalise en minuscules ;
5. conserve les erreurs dans `invalidLines`.

La validation reste donc obligatoire cote main process. Le renderer ne recoit aucun acces SQL ou Node.

## Logique de deduplication

Deux niveaux de deduplication sont appliques :

1. les jobs sont dedupliques dans l'entree utilisateur via leur `jobId` ;
2. chaque slot est cree ou reutilise via `remote_images.remote_url`.

Si un job est colle plusieurs fois, il n'est traite qu'une seule fois.

Si un job existe deja partiellement, seuls les slots manquants sont crees. Les slots existants sont reutilises et leurs informations Midjourney sont conservees ou remises a jour.

## Resultat IPC

Ajout du canal minimal :

```txt
midjourney:add-jobs-batch
```

Resultat renderer expose :

```ts
{
  detectedJobCount: number;
  createdImageCount: number;
  existingImageCount: number;
  invalidLineCount: number;
  jobs: Array<{
    jobId: string;
    createdCount: number;
    existingCount: number;
    imageIds: number[];
  }>;
  invalidLines: Array<{
    line: string;
    reason: string;
  }>;
  images: ImageFile[];
}
```

Ajout aussi d'un canal evenement menu :

```txt
midjourney:add-jobs-batch-requested
```

## Comportement en cas de lignes invalides

Les lignes invalides n'annulent pas l'import complet si au moins un job valide est present.

Apres import, l'interface affiche :

- le nombre de jobs detectes ;
- le nombre d'images creees ;
- le nombre d'images deja presentes ;
- le nombre de lignes ignorees ;
- la liste des lignes ignorees avec leur raison.

Si aucune ligne exploitable n'est presente, l'IPC renvoie une erreur lisible.

## Donnees creees

Pour chaque job valide, Iconotheque cree ou reutilise les quatre URLs :

```txt
https://cdn.midjourney.com/<job_id>/0_0.png
https://cdn.midjourney.com/<job_id>/0_1.png
https://cdn.midjourney.com/<job_id>/0_2.png
https://cdn.midjourney.com/<job_id>/0_3.png
```

Les images sont enregistrees comme images distantes :

- `images.source_kind = 'remote'` ;
- `remote_images.provider = 'midjourney'` ;
- `remote_images.provider_id = <job_id>` ;
- `remote_images.provider_group_id = <job_id>` ;
- `remote_images.remote_slot = 0_0 / 0_1 / 0_2 / 0_3` ;
- `remote_images.source_status = 'remote'`.

## Risques / limites

- Les URLs Midjourney peuvent rester indisponibles selon les restrictions du CDN ; dans ce cas, les tuiles distantes affichent l'erreur visuelle introduite en mission 034.
- Le resume liste les lignes invalides mais ne propose pas encore de correction assistee.
- La deduplication se fait par URL distante exacte, ce qui est adapte au format Midjourney actuel.

## Recommandations pour la suite

Faire une recette manuelle avec :

1. deux jobs valides ;
2. un re-import des memes jobs ;
3. un melange URL + UUID ;
4. une ligne invalide ;
5. verification visuelle dans `Web / Midjourney` ;
6. verification qu'une image importee reste annotable.

Une mission ulterieure pourra envisager une vue groupee par job, mais ce n'est pas necessaire pour valider ce batch minimal.
