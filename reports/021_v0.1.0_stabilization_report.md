# Rapport - Verification et stabilisation v0.1.0

## Resume de la mission

Verification legere de la coherence de la version `0.1.0`, de la documentation de gel, de l'aide integree et des garanties non destructives de la v0.1.0.

La mission n'ajoute aucune fonctionnalite. Les seules corrections appliquees concernent des libelles de version trop courts.

- `docs/DEMO_FREEZE_V0.1.md`

- `reports/021_v0.1.0_stabilization_report.md`

## Verifications effectuees

- Verification de la version `0.1.0` dans `package.json` et `package-lock.json`.
- Verification des libelles de version dans le changelog, la documentation de gel et l'aide integree.
- Relecture de la documentation de gel v0.1.0.
- Relecture du contenu de l'aide integree locale.
- Verification statique du menu `Aide > Aide Iconotheque`.
- Verification statique du raccourci `F1`.
- Verification statique de la fermeture de l'aide par `Echap`.
- Verification statique du bouton `Fermer`.
- Verification des operations potentiellement destructives dans `electron/` et `src/`.
- Verification de `contextIsolation: true` et `nodeIntegration: false`.
- Verification de l'absence de `file://` et `pathToFileURL` dans `electron/` et `src/`.
- `npm.cmd run typecheck`
- `npm.cmd run build`
- Lancement court Electron avec `npm.cmd start`.
- Arret des processus Electron de test.

## Resultats

- La version projet est coherente en `0.1.0`.
- Le changelog, la documentation et l'aide integree affichent maintenant `V0.1.0`.
- La documentation explique clairement :
  - les actions principales ;
  - les raccourcis ;
  - les limites connues ;
  - la separation entre fichiers originaux et metadonnees locales ;
  - le principe non destructif.
- L'aide integree correspond a l'etat reel de la v0.1.0.
- Le menu Aide et le canal IPC `help:requested` sont presents.
- Le raccourci `F1` est declare dans le menu Electron.
- Le composant `HelpPanel` se ferme par bouton et par `Echap`.
- `npm.cmd run typecheck` : reussi.
- `npm.cmd run build` : reussi.
- Le lancement court Electron demarre correctement.
- Aucune occurrence `file://` ou `pathToFileURL` n'a ete trouvee dans `electron/` ou `src/`.

## Anomalies trouvees

- `CHANGELOG.md`, `docs/DEMO_FREEZE_V0.1.md` et `HelpPanel` utilisaient encore le libelle `V0.1` alors que la version applicative est `0.1.0`.
- Aucune anomalie bloquante detectee pendant typecheck, build ou lancement court.

## Corrections realisees

- Remplacement de `V0.1` par `V0.1.0` dans `CHANGELOG.md`.
- Remplacement de `V0.1` par `V0.1.0` dans `docs/DEMO_FREEZE_V0.1.md`.
- Remplacement de `V0.1` par `V0.1.0` dans le titre et la section limites de `HelpPanel`.

## Parcours de demonstration

Parcours relu et verifie statiquement :

- ouverture d'un dossier racine ;
- navigation dans l'arborescence ;
- scan non recursif du dossier actif ;
- affichage de la grille ;
- selection simple et multiple ;
- edition individuelle et par lot ;
- recherche simple / avancee ;
- filtres workflow ;
- visionneuse ;
- aide F1.

Validation effectuee :

- le build de production passe ;
- l'application demarre en lancement court Electron.

Limite :

- le parcours complet avec un vrai dossier d'images n'a pas ete rejoue manuellement dans la fenetre.
- l'ouverture effective du menu Aide et l'appui F1 n'ont pas ete testes manuellement, mais leur cablage a ete verifie dans le code.

## Securite v0.1.0

- Aucune logique de deplacement, renommage, suppression ou modification des fichiers originaux n'a ete identifiee dans le code applicatif.
- Les ecritures reperees concernent la base SQLite locale dans le dossier applicatif Electron.
- Les metadonnees utilisateur restent stockees dans la base locale.
- Les dossiers photo selectionnes ne recoivent pas d'ecriture.
- Les apercus restent servis via `iconotheque-image://`.
- Le renderer reste isole de Node.

## Recommandations pour la suite

- Faire une recette manuelle courte avec un dossier d'images reel avant toute demonstration.
- Verifier manuellement `Aide > Aide Iconotheque`, `F1`, `Fermer` et `Echap`.
- Garder la v0.1.0 comme base stable avant de demarrer une v0.2.
- Pour v0.2, choisir une seule evolution structurante a la fois : EXIF, hash fichier, miniatures, FTS ou packaging.
