# Mission 039 - 0.1.3-dev.13 - Collections virtuelles V0

## Resume

Mission realisee depuis `0.1.3-dev.12`, avec passage en `0.1.3-dev.13`.

Objectif : ajouter une premiere version de collections virtuelles internes, basees sur `images.id`, capables de contenir des images locales et distantes sans modifier les fichiers originaux.

## Tables creees

Ajout de `collections` :

- `id INTEGER PRIMARY KEY`
- `name TEXT NOT NULL UNIQUE`
- `description TEXT`
- `created_at TEXT`
- `updated_at TEXT`

Ajout de `collection_images` :

- `collection_id INTEGER NOT NULL`
- `image_id INTEGER NOT NULL`
- `added_at TEXT`
- `sort_order INTEGER`
- `PRIMARY KEY(collection_id, image_id)`
- `FOREIGN KEY(collection_id) REFERENCES collections(id) ON DELETE CASCADE`
- `FOREIGN KEY(image_id) REFERENCES images(id) ON DELETE CASCADE`

## Migration ajoutee

`SCHEMA_VERSION` passe de `6` a `7`.

La migration cree les tables avec `CREATE TABLE IF NOT EXISTS`.

Elle ne supprime pas d'image, ne modifie pas `local_images`, ne modifie pas `remote_images` et ne touche pas aux metadonnees utilisateur.

## Affichage colonne gauche

`FolderTree` affiche une section `Collections` sous les sources Web.

Chaque collection affiche :

- son nom ;
- son compteur d'images ;
- une icone visuellement distincte des dossiers et sources Web.

Un bouton compact `+` permet d'ouvrir la modale de creation.

## Creation d'une collection

Ajout de `Fichier > Nouvelle collection...`.

La modale minimale demande un nom.

Validation :

- nom vide refuse ;
- doublon refuse via contrainte `UNIQUE(name)` et message lisible ;
- succes : la collection est creee, la liste est rechargee et la collection est selectionnee.

## Ajouter la selection a une collection

Ajout de `Edition > Ajouter la selection a une collection...`.

La commande est activee uniquement quand :

- au moins une image est selectionnee ;
- au moins une collection existe.

La modale permet de choisir une collection existante.

Le main process insere dans `collection_images` uniquement les couples absents.

Message :

```txt
Ajout termine : X image(s) ajoutee(s), Y deja presente(s).
```

## Affichage des images d'une collection

Cliquer une collection :

- bascule la source active sur `collection` ;
- charge les images via `collection_images.image_id` ;
- reconstruit des `ImageFile` compatibles avec la grille existante ;
- permet selection, multi-selection, visionneuse, panneau infos et metadonnees.

Les images locales et distantes peuvent cohabiter dans la meme collection.

## Dedoublonnage

Le dedoublonnage repose sur :

```txt
PRIMARY KEY(collection_id, image_id)
```

Le main process compte aussi les images deja presentes avant insertion pour fournir un message utilisateur clair.

## Retrait d'une image de collection

Disponible via :

```txt
Edition > Retirer la selection de la collection
```

La commande est activee uniquement quand :

- la vue courante est une collection ;
- au moins une image est selectionnee.

Le retrait supprime uniquement des lignes `collection_images`.

Il ne supprime pas :

- l'image du catalogue ;
- le fichier local ;
- `local_images` ;
- `remote_images` ;
- les metadonnees utilisateur.

## Renommage / suppression collection

Non disponible dans cette V0.

Ces actions restent a cadrer dans une mission ulterieure.

## Confirmation images.id

Les collections utilisent exclusivement `collection_images.image_id`, lie a `images.id`.

Elles ne pointent ni vers `local_images.path`, ni vers `remote_images.remote_url`.

## Confirmation fichiers originaux

Aucun fichier original n'est modifie, copie, deplace, renomme ou supprime.

La mission n'ecrit pas dans les dossiers photo.
