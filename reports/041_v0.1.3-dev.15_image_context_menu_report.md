# Mission 041 - 0.1.3-dev.15 - Menu contextuel images et collections

## Resume

Mission realisee depuis `0.1.3-dev.14`, avec passage en `0.1.3-dev.15`.

Objectif : ajouter un menu contextuel non destructif sur les images de la grille afin de rendre les actions de collections plus directes.

La recette utilisateur de `0.1.3-dev.14` etant indiquee OK, cette mission s'appuie sur les handlers existants d'ajout et de retrait de collection.

## Fichiers modifies

- `package.json`
- `package-lock.json`
- `CHANGELOG.md`
- `src/App.tsx`
- `src/components/ImageGrid.tsx`
- `src/styles/app.css`

## Approche choisie

Approche retenue : menu contextuel React/CSS cote renderer.

Raisons :

- implementation plus simple et locale a l'interface ;
- pas de nouveau canal IPC ;
- pas d'acces SQL depuis le renderer ;
- reutilisation directe des flux existants ;
- controle simple de l'affichage selon le contexte.

## Actions ajoutees

Le clic droit sur une tuile d'image ouvre un menu contextuel avec :

- `Ouvrir dans la visionneuse` ;
- `Ajouter a une collection...` ;
- `Retirer de cette collection` uniquement dans une vue collection.

Aucune action destructive n'a ete ajoutee.

## Regles d'activation

`Ajouter a une collection...` est desactive si :

- aucune image n'est selectionnee ;
- aucune collection n'existe.

`Retirer de cette collection` est visible uniquement si :

- la vue active est une collection ;
- la recherche avancee globale n'est pas en train de remplacer la grille.

La disponibilite du menu `Edition > Retirer la selection de la collection` a ete alignee sur cette meme regle pour eviter un retrait depuis une grille de recherche globale.

## Selection au clic droit

Le comportement suit les conventions habituelles :

- clic droit sur une image deja selectionnee : la selection multiple existante est conservee ;
- clic droit sur une image hors selection : la selection devient cette seule image.

La tuile ciblee devient aussi l'image courante afin que `Ouvrir dans la visionneuse` ouvre bien l'image cliquee.

## Multi-selection

La multi-selection est conservee quand le clic droit porte sur une image deja selectionnee.

Les actions `Ajouter a une collection...` et `Retirer de cette collection` reutilisent ensuite `selectedImageIds`, comme les menus applicatifs existants.

## Reutilisation de l'ajout a collection

Le menu contextuel appelle le meme handler renderer que :

- `Edition > Ajouter la selection a une collection...` ;
- `CommandOrControl+Shift+C`.

La modale existante est donc reutilisee, avec les messages de `0.1.3-dev.14` :

- image ajoutee ;
- images ajoutees ;
- deja dans la collection ;
- ajout partiel.

La logique d'insertion et de dedoublonnage reste cote main process via les IPC existants.

## Reutilisation du retrait de collection

Le menu contextuel appelle le meme handler renderer que :

- `Edition > Retirer la selection de la collection`.

Le retrait supprime uniquement les associations `collection_images`.

Il ne supprime pas :

- l'image du catalogue ;
- les fichiers locaux ;
- `remote_images` ;
- les metadonnees utilisateur.

## Fermeture du menu

Le menu se ferme :

- apres choix d'une action ;
- au clic exterieur ;
- au clic droit exterieur ;
- avec `Echap` ;
- si l'image cible disparait de la grille ;
- lors d'un changement de source/vue.

Le positionnement est contraint dans la fenetre avec une marge minimale.

## Recette manuelle

La recette interactive complete n'a pas ete effectuee dans cette session.

Non verifie manuellement :

- clic droit sur image non selectionnee ;
- clic droit sur image deja selectionnee dans une selection multiple ;
- ajout a collection via menu contextuel ;
- ajout multiple via menu contextuel ;
- message de doublon via menu contextuel ;
- retrait depuis une vue collection via menu contextuel ;
- fermeture visuelle du menu au clic exterieur et apres action.

## Limites restantes

- Pas de drag and drop vers une collection.
- Pas de menu contextuel sur les collections elles-memes.
- Pas de renommage de collection.
- Pas de suppression de collection.
- Pas de feedback global persistant apres retrait ; le refresh visible de la grille reste le feedback V0.
- L'action `Modifier les metadonnees par lot...` n'a pas ete ajoutee au menu contextuel pour garder cette premiere version compacte.

## Securite

Le menu contextuel ne declenche aucune action destructive.

Aucun fichier image original n'est modifie, copie, deplace, renomme ou supprime.

La mission n'ecrit pas dans les dossiers photo.

## Recommandations pour la suite

- Faire une recette visuelle sur un vrai dossier avec une collection contenant plusieurs images.
- Ajouter plus tard un feedback sobre apres retrait si un systeme de notifications global est introduit.
- Cadrer separement un drag and drop vers collection, car cela implique des etats de survol et de destination.
