# Mission 036 - 0.1.3-dev.10 - UX import web/Midjourney et messages de doublons

## Resume

## Messages UX ajoutes

Les messages d'import Midjourney unitaire sont maintenant explicites :

- job nouveau : `Job Midjourney ajoute : 4 images creees.` ;
- job deja present : `Job Midjourney deja present : aucune image creee, 4 images deja presentes.` ;
- job partiellement present : `Job Midjourney complete : X image(s) creee(s), Y deja presente(s).`

Pour l'import par lot, le resume affiche toujours :

- jobs detectes ;
- images creees ;
- images deja presentes ;
- lignes ignorees.

Exemple de forme :

```txt
Import termine : 2 job(s) detecte(s), 4 image(s) creee(s), 4 deja presente(s), 1 ligne(s) ignoree(s).
```

Les lignes invalides restent affichees dans le bloc dedie de la modale.

## Signalement des doublons

Les doublons ne sont pas seulement deduits par les compteurs techniques.

Cas couvert :

- si les 4 slots existent deja, le message indique explicitement que le job est deja present ;
- si certains slots existent deja, le message indique que le job a ete complete ;
- en batch, le resume global affiche le total des images deja presentes.

La logique de deduplication reste celle existante :

- par `jobId` pour les jobs saisis en batch ;
- par `remote_images.remote_url` pour chaque slot.

## Detection Midjourney dans l'ajout web

Dans `electron/main.ts`, `addRemoteImageFromUrl(...)` valide d'abord l'URL distante.

Ensuite :

1. si le hostname est `cdn.midjourney.com`, l'URL est validee avec la logique Midjourney existante ;
2. si l'URL Midjourney est invalide, elle est refusee ;
3. si elle est valide, le main process appelle `createOrReuseMidjourneyJob(...)` ;
4. le resultat IPC revient avec `route: "midjourney"` ;
5. le renderer bascule vers `Web / Midjourney` et affiche un message dedie.

Ainsi, une URL Midjourney collee dans `Ajouter une image web...` ne cree plus d'image `generic_url`.

## Reutilisation de la logique existante

La mission reutilise la logique creee precedemment :

- `validateMidjourneyInput(...)` pour reconnaitre une URL CDN Midjourney ;
- `createOrReuseMidjourneyJob(...)` pour creer ou reutiliser les quatre slots ;
- `remote_images.remote_url` pour eviter les doublons ;
- `updateRemoteImageProviderDetails(...)` pour requalifier un slot deja present.

Si une URL Midjourney avait ete importee auparavant comme `generic_url`, le slot correspondant est requalifie en `midjourney` lorsqu'il est rencontre par le parcours Midjourney.

## Comportement pour les URLs generiques

Les URLs non Midjourney conservent le comportement existant :

- validation HTTPS ;
- creation ou reutilisation d'une image distante ;
- `provider = 'generic_url'` ;
- affichage dans `Web / Images par URL`.

## Comportement pour les URLs Midjourney

Les URLs `https://cdn.midjourney.com/<job_id>/<slot>.png` :

- ne sont plus ajoutees comme images web generiques ;
- creent ou reutilisent les 4 slots du job ;
- utilisent `provider = 'midjourney'` ;
- apparaissent dans `Web / Midjourney` ;
- affichent un message clair si le job est nouveau, deja present ou partiellement present.

Les URLs Midjourney mal formees sont refusees au lieu de retomber en image generique.

## Limites restantes

- Pas de notification globale hors modale.
- Pas de vue groupee par job.
- Pas d'extraction de prompt.
- Pas de cache, archive locale ou telechargement persistant.
- Pas de connexion Midjourney, API Midjourney ou scraping.

## Confirmation securite

Aucun fichier image original n'est modifie, deplace, renomme ou supprime.

La mission ne cree pas de cache, n'ecrit pas dans les dossiers photo et ne modifie pas le protocole controle `iconotheque-image://`.

## Recommandations pour la suite

Faire une recette utilisateur courte sur les quatre parcours :

1. job Midjourney deja present ;
2. batch mixte avec ligne invalide ;
3. URL Midjourney collee dans `Ajouter une image web...` ;
4. URL generique collee dans `Ajouter une image web...`.

Si les messages sont juges utiles, une future mission pourra ajouter une zone de notification globale discrete pour eviter de garder les modales ouvertes apres succes.
